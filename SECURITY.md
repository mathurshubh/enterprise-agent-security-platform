# Security Policy

> **Project Status**
>
> This repository is a research and reference implementation intended to demonstrate secure architecture patterns for governing enterprise AI agents. While it implements production-inspired design patterns, it is not intended for direct deployment in production environments. Some enterprise capabilities—including persistent storage, distributed execution, observability, and operational hardening—are intentionally outside the current project scope.
>
> Security vulnerabilities are still taken seriously and should be reported responsibly using the process described below.

## Supported Versions

| Version | Supported |
|----------|-----------|
| main     | ✅ |
| v0.8.x   | ✅ |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, use one of the following:

- GitHub Security Advisories (preferred)
- Email: <mathurshubhankar@gmail.com>

Include:
- Description
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)

## Response Targets

Initial response: 72 hours

Status update: Within 7 days

Fix timeline: Depends on severity.

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing a vulnerability.

## Scope

This policy applies to:

- Enterprise Agent Security Platform
- Runtime Security Pipeline
- Provider Integrations
- Policy Engine
- Authorization Engine
- Detection Engine
