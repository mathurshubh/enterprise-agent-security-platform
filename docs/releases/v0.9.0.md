# Enterprise Agent Security Platform v0.9.0

Release date: July 4, 2026

---

## Overview

Version 0.9.0 transitions the Enterprise Agent Security Platform from basic runtime execution toward a governed enterprise tool ecosystem.

The release preserves the platform's Zero Trust architecture while adding the governance structures required to manage executable tools as first-class enterprise security assets. LLMs remain untrusted intent parsers. Authorization, policy evaluation, detection, risk assessment, response, audit logging, and tool execution remain deterministic platform responsibilities.

The primary goal of v0.9.0 is to establish a governed tool layer built around the Tool Registry, immutable Tool Metadata, the `BaseTool` abstraction, metadata-only discovery, metadata-only inventory, and runtime-controlled tool resolution.

## New Features

### Tool Governance

v0.9.0 introduces a governed tool model for executable enterprise capabilities.

- Centralized Tool Registry for registered executable tools.
- Rich Tool Metadata describing tool identity, capabilities, governance attributes, and operational characteristics.
- `BaseTool` abstraction for executable platform tools.
- Runtime resolution of executable tools through the Tool Registry.
- Tool Discovery exposing metadata without executable tool instances.
- Tool Inventory exposing registered Tool Metadata for read-only inventory use cases.

### Runtime

The runtime now separates provider integration, intent parsing, deterministic security evaluation, and secure tool execution.

- `ProviderFactory` centralizes configured provider construction.
- `ProviderAdapter` isolates provider-specific integrations.
- `EnterpriseAgent` converts natural language requests into structured `ToolInvocation` objects.
- Runtime execution pipeline evaluates `ToolInvocation` objects through deterministic authorization, policy evaluation, detection, risk assessment, response selection, and Tool Registry resolution.

### Security

The release preserves deterministic security enforcement while extending the runtime around governed tools.

- Deterministic authorization.
- Resource-aware authorization.
- Risk assessment through `RiskAssessment`.
- Detection through deterministic finding generation.
- Response selection based on assessed risk.
- Audit logging and audit-relevant runtime events.

### Providers

The provider layer remains isolated from security decisions.

- Ollama provider support.
- Gemini provider support.
- Provider selection through the Provider-agnostic Architecture.

## Architectural Improvements

### Provider Integration and Governance Separation

Provider-specific behavior is isolated behind `ProviderAdapter` implementations and selected through `ProviderFactory`. Providers produce structured intent, but they do not authorize, evaluate policy, assess risk, select responses, or execute tools.

### Centralized Tool Registry

Executable tools are registered and resolved through the centralized Tool Registry. Runtime components do not instantiate or execute tools directly.

### Immutable Tool Metadata

Tools expose immutable Tool Metadata describing identity, capability, governance, and operational characteristics. Discovery and inventory surfaces expose metadata only and do not expose executable `BaseTool` instances.

### Runtime Security Pipeline

Every `ToolInvocation` is treated as untrusted input and evaluated through deterministic runtime controls before executable tool resolution. The pipeline preserves authorization, policy evaluation, detection, risk assessment, response, and auditability.

### Zero Trust Architecture

The platform continues to treat user prompts, provider responses, `ToolInvocation` objects, tool outputs, and external content as untrusted. Security decisions remain deterministic and platform-owned.

## Breaking Changes

There are no breaking public API changes in v0.9.0.

The internal runtime architecture has evolved significantly. Tool execution is now governed through the Tool Registry and `BaseTool` abstraction, with Tool Metadata used for governance, discovery, and inventory.
